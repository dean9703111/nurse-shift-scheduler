# 護理師排班產生器（純前端 · 離線單檔）

上傳「上月 Excel 班表」，一鍵產生「下月班表」。整個工具是**單一 HTML 檔、雙擊即用**，不需安裝、不需 Python、不需後端；班表資料**全程留在本機瀏覽器、不上傳**。

> 輸出檔以上傳檔為範本，**格式、樣式、COUNTIF／驗證公式完整保留**；日期、星期、國定假日依真實日曆自動產生。

## 特色

- 📄 **單一檔案離線可用**：`index.html` 內嵌 ExcelJS 與排班邏輯，雙擊即可在 Chrome / Edge / Safari 開啟。
- 🗓️ **自動推算下一週期**：讀取上傳檔的年月與視窗長度，整體往後推一個月自動預填（例：上傳 5/10–6/30 → 產生 6/10–7/31）。
- 🧮 **保留公式與格式**：只覆寫排班資料格，範本的統計／驗證公式原封不動。
- 🇹🇼 **國定假日自動標註**：內建 2026 年假日表，能連網時抓取最新資料（[ruyut/TaiwanCalendar](https://github.com/ruyut/TaiwanCalendar)）覆蓋；只標視窗實際涵蓋到的假日。
- 👥 **人員 × 日期完整格線**：依階層分組、上月對照區塊、違規紅色標示、每日人力摘要。
- 🛌 **預先請假**：自動載入所有人員，可個別設定預假日並重新產生。
- ✅ **硬規則驗證**：最低人力、階層上限、小組長、連續上班/大夜、換班休息（11 小時）、N-off-D（大夜後僅休 1 天不得接白班）、休假視窗，違規即時標示。
- 🧑‍🎓 **新人不排班**：預假面板可勾選「不排班」，整月排除該員（輸出留白、不計入人力）。

## 使用方式

1. 下載或 clone 本 repo，直接**雙擊 `index.html`**（或見下方 GitHub Pages）。
2. 上傳上月班表 `.xlsx`。
3. 確認自動預填的年月、起始日與規則。
4. （可選）在「預先請假設定」填入個別人員預假日。
5. 按「產生下月班表」→ 檢視結果 → 下載。

### GitHub Pages（線上版）

本 repo 已是靜態網頁，可於 **Settings → Pages → Deploy from branch → main / root** 啟用，之後即可用網址開啟 `index.html`。

## 開發

原始碼在 `src/`，`index.html` 是打包後的成品。

```
├── index.html          # 打包後的單檔應用（可直接用）
├── build.sh            # 重新打包 index.html
├── package.json
└── src/
    ├── head.html       # 介面 HTML / CSS
    ├── scheduler.js    # 排班核心：解析 / 產生 / 驗證 / 寫回 / 日曆 / 假日
    ├── app.js          # 介面控制器
    ├── test_node.js    # 核心邏輯測試（Node）
    ├── test_html.js    # 單檔 HTML 瀏覽器模擬測試（jsdom）
    └── test_rules.js   # 需求規則驗證：規則常數對照範本公式＋驗證器正反例＋產生結果獨立稽核
```

重新打包與測試：

```bash
npm install          # 安裝 exceljs、jsdom
npm run build        # 重新產生 index.html
npm test             # 執行測試（需在根目錄放一份 sample.xlsx）
```

> 測試需在專案根目錄放符合格式的班表樣本。基於個資考量，實際班表**不會**入庫（`.gitignore` 以 `*.xlsx` 一律排除）。已知**兩種上傳情境**各備一份樣本、皆通過驗證，日後測試直接沿用即可：
>
> | 樣本檔 | 上傳情境 | 特徵 |
> |--------|----------|------|
> | `sample.xlsx` | 跨月窗口表（本工具產生的下載檔） | 起始非 1 號、視窗橫跨兩個月（如 5/10–6/30）；產生下月時前段與上傳檔重疊，成灰色「銜接段」 |
> | `sample-monthly.xlsx` | 完整單月表（醫院原始格式） | 1 號到月底（如 7/1–7/31）；與新月份無重疊（`carryLen=0`），畫面改以「上月末對照」呈現跨月銜接 |
>
> `npm test` 的瀏覽器模擬用 `sample.xlsx`；單月情境可另以 `sample-monthly.xlsx` 手動驗證（兩者的跨月銜接、休假留空、驗證↔獨立稽核交叉比對均一致）。

## 技術

| 功能 | 技術 |
|------|------|
| 讀寫 xlsx（保留公式+樣式） | [ExcelJS](https://github.com/exceljs/exceljs)（內嵌） |

## 開發歷程：踩過的坑與優化

以下是本工具在多輪實測迭代中遇到的問題與對應解法，供後續維護者參考——多數是「症狀看似 A、根因其實在 B」的陷阱。

### 排班演算法的三次翻修：工作量平衡

「讓每個人天數盡量平均」看似簡單，實際來回震盪了三個版本才收斂，教訓值得記錄：

| 版本 | 排序策略 | 結果（7 月實測） |
|------|----------|------------------|
| v1 初版 | 同條件偏向名單前面 + 「已上過該班者優先」 | A 階平均 22.4 天（最高）、E 階 14.1 天，**E13/E14 整月 0 班**，範圍 0~23 |
| v2 資淺優先 | 資淺者（E→A）優先、資深補位 | E 階拉到 20.1 天，但**未包班的資深幾乎沒班**（最少 1 天） |
| v3 包班優先 | 把「包班者優先」放最前 | 包班者被每天選走**排滿 23 天**、未包班者吃剩只有 14 天 |
| v4 定案 | **工作天數少者絕對優先**為唯一核心 | 全體 71 人落在 17~22 天，休假差距 ≤2 天，**0 違規** |

> **關鍵領悟：包班只該是「限制」（只能上那種班別），不該是「優先權」。** 一旦把包班或資淺當成優先權，就會把某群人排滿、另一群人餓死。定案版把排序退回「工作天數少者優先」這唯一主軸，包班者休假後工作量自然變低、下一輪被選回上他的班，不需要任何特權；班別相容、資淺、六日公平都只在「同工作量」時才作為次要微調。

### 換班與休假規則的補全

- **N→E 換班休息不足**：原本只禁止「N/E 後接 D」，漏了 N→E（大夜值到隔日 08:00，接晚班只休 8 小時）。改為通用的換班禁止矩陣 `forbidTransition`（`N→D`、`N→E`、`E→D` 皆禁），`generate` 排班時直接避開、`validate` 標紅並註明「休息不足 11 小時」，含銜接段第一天。
- **N-off-D 缺漏**（大夜後休 1 天不得接白班）：原實作只擋相鄰換班，沒擋「隔一天」的情形。以真實班表佐證解讀——歷史資料中 N-off-E、E-off-D 都有出現，**唯獨 N-off-D 是 0 次**，證明醫院確實視為禁止模式，才固化進規則。
- **休假違規幾乎必炸**：排班器原本只顧「填滿每日人力」、從不主動預留休假，滑動視窗（2 週 2 例假、4 週 8 休）幾乎必違規。加入建構式的**休假保底** `mustRest`：每天排班前檢查某人「再不休今天就註定違反視窗」則強制休假，保證不違規而非事後補救；門檻放入 UI 彈性規則，設 0 即停用。
- **每月至多 2 種班別**（A2 一度被排到 3 種）：新增 `maxShiftKinds`，一旦用滿 2 種就不再開第三種。
- **六日休假不平均**：原本完全沒處理週末公平，有人整月週末 0 休。加入「週末上班少者優先上班」排序，並在該月最後一個週末日仍為 0 時**強制休假**保底，實測人人至少休 1 個週末日。

### Excel 讀寫的坑（最隱蔽）

- **⚠️ 最大的坑：重疊月份是「重新亂排的假資料」**。產生下一週期時，畫面上與上傳檔重疊的那段（如 6/10~6/30）原本是程式**重新亂排、完全沒沿用上傳檔實際班表**的假資料。新增 `buildCarry`：把新視窗與上傳檔重疊的天數逐格對齊真實日曆後直接沿用原始內容（含特、國、公、空白），下載的 Excel 也原樣寫回，只有新月份是新產生的；銜接段是既成事實，不再誤報違規。
- **ExcelJS 樣式物件共用污染**：範本儲存格會**共用同一個樣式物件**，直接改某格的 `fill` 會連帶污染其他格。必須複製樣式再改。
- **底色規則反覆修正**：一度以為「所有例/息都要標黃」，實際上需求只有**預排假**標黃、演算法排出的休假不標色；上月銜接段一律不標黃；日期列則依月份自動上色（上月段灰底、本月段淡藍），分界跟著銜接段長度自動計算。
- **底部驗證區沒同步**：範本在人員名單下方另有一列日期（供 COUNTIF 對照）與其底色，`writeCalendar` 起初只改了最上面那列，導致上下兩列日期／換色點不一致。改為通用掃描所有標「日期」的列一併同步，人數增減使驗證區位移也能自動找到。
- **`classifyCell` 前綴語意**：範本的 COUNTIF 用 `"D*"` 前綴判斷上班，原程式用「包含」判斷會把「病E」「生理d」誤判為上班。改為前綴判斷，與範本一致。

### 跨月銜接：兩種上傳情境都要成立

上傳檔可能是**兩種格式**，「月底→月初不可 N 接 D」的跨月銜接在兩者都須正確（樣本見上方測試表）：

| 上傳情境 | 銜接段 | 上月末呈現 |
|----------|--------|-----------|
| 跨月窗口表（起始非 1 號，如 6/10–7/31） | 有（新視窗與上傳檔重疊段，灰色照抄、寫回 Excel） | 銜接段本身即含上月末 |
| 完整單月表（1 號到月底，如 7/1–7/31） | 無（`carryLen=0`） | 新增畫面「上月末對照」（唯讀灰階、僅畫面不寫 Excel） |

- **檢查邏輯本來就跨月成立**：`generate` 排新月第一天時會讀上月最後一天（`tail`）避開 N→D；`validate` 對 `carryLen=0` 的首日也會標紅（「上月末(N)後首日排D」）。單月表「看起來沒銜接」的真因是**畫面看不到上月末**——故補上「上月末對照」段（`app.js` `renderGrid` 的 `showTail`），讓 7/31→8/1 一眼可核對；下載的 Excel 仍是純新月份（欄位數固定，放不下額外對照）。
- **銜接段內的跨月邊界也要抓**：原本銜接段整段視為既成事實不報，但若跨月邊界落在銜接段內仍可能藏著 N→D。`validate` 改為依真實日曆月份判斷——銜接段內「同月」才跳過，「跨月」邊界（月底→月初）一律檢查；無月份資料（純邏輯測試）時維持原行為。
- **新月份休假改為留空**：`labelOffs` 原本自動把休假標「例／息」（每 7 天首日例、其餘息）。改為**一律留空白**，由護理長依實際狀況（例假與休息日之法定區分）自填；銜接段照抄上傳檔原始內容不受影響。

### UI／UX 優化

- **違規原因看不到**：原生 `title` 提示延遲近 1 秒、紅色日期表頭甚至完全無提示。改成自訂浮動 tooltip，即時顯示、跟隨滑鼠、自動避開視窗邊緣，涵蓋紅色日期表頭／班別格／人員姓名／每日摘要四種違規位置。
- **銜接段捲動重疊**：淺色效果原用 `opacity:.55`，連固定表頭也變半透明，往下捲時下方格子會「透」上來無法閱讀。改用不透明的白色覆蓋層做出同樣淺色，表頭保持完全不透明。
- **一律改用真實日期**：違規訊息、每日人力摘要、預假輸入從「第 N 天」的天序改為真實日期（7/12、填 `5,6` 即本月 5、6 日），摘要也只從實際排班月開始顯示。
- **錯誤訊息明確化**：從「人數不夠」細化到「哪個班別缺幾人」（如「D 班僅 22 人，低於最低人力 30 人（缺 8 人）」）。

### 已知限制

- **階層上限門檻寫死**：`tierCaps`（B-E≤14/14/12、C-E≤11/11/9…）目前是照範本抄進預設規則、UI 也無欄位可調。若護理長某月調整了 Excel 驗證列的門檻，產生器不會自動跟著變。理想做法是上傳時直接從 Excel 驗證列的「D14」「N12」標籤讀出門檻。
- **個人統計公式欄位範圍寫死**：範本右側的個人月統計（例假日/四週例等）公式假設月份分界固定在某欄，銜接段長度不同（如 22 天 vs 21 天）時，下載檔這些**跨月統計會偏移一天**，數字不能盡信；底部驗證列是逐欄計算、不受影響。

### 驗證機制

需求規則以獨立驗證程式 `src/test_rules.js` 把關（納入 `npm test`），四層防線共 80+ 項檢查：規格常數對照範本 Excel 公式、驗證器正反例（含跨月邊界）、與 `validate()` 完全獨立的端對端稽核交叉比對、真實歷史資料佐證規則解讀。
